Purpose
- Simulation-based binary Genetic Algorithm (GA) for air-corridor edge orientation optimization.
- Decision variables are binary genes mapped to corridor edge groups:
  - 0 -> forward edges
  - 1 -> reverse edges

Core objective and validity rules
- Per-seed fitness contribution:
  fitness_seed = collisions + no_path_count + timeout_count
- Chromosome fitness over K seeds:
  fitness = sum(collisions over seeds) + sum(no_path_count over seeds) + sum(timeout_count over seeds)
- Invalid-pressure metric:
  invalid_count = no_path_count + timeout_count
- Invalid flag:
  is_invalid = (invalid_count >= invalid_threshold)
- Best-individual replication dispersion metric:
  best_rep_std = std(per_seed_fitness list)

GA operators and internal behavior
- Selection: tournament selection (size = --tournament-size), minimizing fitness.
- Crossover: uniform crossover with probability --crossover-prob.
- Mutation: bit-flip mutation per gene with:
  mutation_prob = clip(1 / num_variables, 0.002, 0.03)
- Replacement: elitist generational replacement.
  - elites = best --elitism individuals
  - offspring count = population - elitism
  - safe fully-elitist carry-over when offspring count <= 0
- Early stopping:
  stop when no global-best fitness improvement for --early-stop-patience generations.

Replication / seed schedule (CRN protocol)
- Same seed list is used across all individuals inside the same generation.
- All generations:
  - All individuals are evaluated with K=3 seeds.
  - No post-evaluation top-percentile refinement is applied.

Terminal line interpretation (per generation)
- Printed metrics include:
  best, mean, std, invalid count, route-failure means, best_seed_scores, best_rep_std, diversity, generation time.
- best_seed_scores format:
  [seed_id:fitness_seed, seed_id:fitness_seed, ...]
- Number of printed seed entries:
  - 3 entries (best individual is always evaluated with 3 seeds)
- Post-phase terminal progress lines:
  - Final validation prints `[FinalVal]` start/progress/end with per-candidate timing and fitness
  - Sensitivity prints `[Sensitivity]` start/progress/end with periodic updates (every 10 bits) and ETA-like remaining estimate
  - If sensitivity is disabled, prints explicit skipped message

Final-stage validation and sensitivity
- Candidate set: final population + global-best archive candidate.
- Ranking for shortlist: archived best-known normalized selection score.
- Final validation:
  - top `--final-validation-top-k` candidates (default 5)
  - held-out seed set K=`--final-validation-seeds` (default 20)
- Sensitivity:
  - enabled by default; disable with `--no-run-sensitivity`
  - one-bit flip sweep over best validated chromosome
  - evaluates delta fitness per bit on held-out seeds
  - bit sweep cap via `--sensitivity-max-bits` (0 means all bits)

Cache and parallel evaluation behavior
- Cache key:
  (chromosome_bitstring, seed_signature)
- Duplicate chromosomes in same batch are de-duplicated.
- Parallel batch evaluation via thread pool with --workers.
